================================================================================
PACKAGE PURPOSE
================================================================================

This package enables deployment of IBM API Connect v12.1.0.1 in:
  ✓ Airgapped/offline environments
  ✓ Kubernetes 1.24-1.30
  ✓ With Contour ingress controller (v1.28+)
  ✓ Using Harbor container registry

ASSUMES:
  - Contour already installed and configured
  - Harbor registry accessible
  - API Connect images loaded to Harbor (see HARBOR-SETUP.md)
  - Storage class available
  - DNS configured (10 entries or wildcard)

================================================================================
CONFIGURATION SYSTEM
================================================================================

Placeholder-Based Configuration:
  - NAMESPACE_PLACEHOLDER         → Your namespace
  - DNS_PLACEHOLDER               → Your DNS domain
  - INGRESS_CLASS_PLACEHOLDER     → Your ingress class
  - STORAGE_CLASS_PLACEHOLDER     → Your storage class
  - REGISTRY_PLACEHOLDER          → Your registry path
  - REGISTRY_SECRET_PLACEHOLDER   → Your registry secret name

Single Command Configuration:
  One sed command replaces all placeholders across all files.
  See 00-CONFIGURE.txt for details.

================================================================================
DEPLOYMENT WORKFLOW
================================================================================

Phase 1: PREPARATION (Harbor + DNS)
  1. Load API Connect images to Harbor registry (HARBOR-SETUP.md)
  2. Configure 10 DNS entries (DNS-ENTRIES.txt)

Phase 2: CONFIGURATION (5 minutes)
  1. Set environment variables
  2. Run sed command to replace placeholders
  3. Verify configuration

Phase 3: DEPLOYMENT (45-75 minutes)
  1. Create namespace and secrets
  2. Deploy cert-manager (if needed)
  3. Deploy API Connect operators
  4. Deploy 4 subsystems (Management, Gateway, Portal, Analytics)
  5. Deploy HTTPProxy resources

Phase 4: CONFIGURATION (10 minutes)
  1. Access Cloud Manager
  2. Register services
  3. Create provider organization
  4. Configure catalogs

Total Time: ~2 hours for complete deployment
